# CabloyJS

CabloyJS 不是独立的框架，而是通过`Vona套件`和`Zova套件`的形式提供一组开箱即用的高级功能模块，加速全栈项目的开发

## VonaJS、ZovaJS和CabloyJS的关系

1. **VonaJS**：提供后端底层架构及 SSR 适配层，支持前后端分离和类型共享
2. **ZovaJS**：提供前端底层架构及 SSR 渲染能力，结合 Vue3 响应式、React TSX 和 Angular IOC 的优势
3. **CabloyJS**：在 VonaJS 和 ZovaJS 基础上，提供更多开箱即用的高级功能，加速全栈项目的开发

## 研发历程

### 1. 2016年：V1-V4

2016 年启动 CabloyJS 开发。CabloyJS 最初基于 JavaScript 的全栈 NodeJS 框架，经过 V1、V2、V3、V4 等大版本的演进，不断完善架构

- 有网友评价：`CabloyJS 是教科书式的框架`
- 也有建议：`如果 CabloyJS 支持 TypeScript，并采用前后端分离，将更加完美`

### 2. 2023年: V5研发

2023 年，经过充分评估和验证，CabloyJS V5 采用全新架构理念进行彻底重构。基于 TypeScript 开发，采用前后端分离架构，孵化出两个底层框架：

- **ZovaJS**：直观的前端框架 = Vue3 响应式 + React TSX + Angular IOC
- **VonaJS**：全栈框架，支持单代码库构建`SSR/SPA/Web网站/Admin中后台`，内置前后端类型共享能力

### 3. 2026年：V5发布

2026 年，ZovaJS V5 和 VonaJS V5 正式发布。CabloyJS V5 在全新底层框架之上，不断提供更多高级能力，进一步提升开发体验和效率，践行“教科书式框架”的愿景和使命。
